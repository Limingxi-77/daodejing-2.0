道德经平台 - 前后端启动说明

一、启动前准备

1. 打开小皮面板，启动 MySQL。
2. 确认后端数据库配置文件存在：
   backend\.env
3. 确认 backend\.env 中 MySQL 配置和小皮一致：
   MYSQL_HOST=localhost
   MYSQL_PORT=3306
   MYSQL_USER=daodejing
   MYSQL_PASSWORD=change-me
   MYSQL_DATABASE=daodejing_platform
4. 如果首次运行，先安装依赖。在项目根目录打开 PowerShell，执行：
   cd "C:\Users\32415\OneDrive\Desktop\daodejing-2.0-main"
   npm --prefix backend install
   npm --prefix vue-project install

二、启动后端

打开第一个 PowerShell 窗口，执行：

cd "C:\Users\32415\OneDrive\Desktop\daodejing-2.0-main"
npm run backend:dev

启动成功后，后端会显示类似内容：

MySQL 连接成功
hAdmin 静态后台入口: http://localhost:8000/hadmin/login.html
前台业务后端运行在 http://localhost:8000
JWT 用户认证已就绪

后端地址：
http://localhost:8000

后台管理地址：
http://localhost:8000/hadmin/login.html

三、启动前端

再打开第二个 PowerShell 窗口，执行：

cd "C:\Users\32415\OneDrive\Desktop\daodejing-2.0-main"
npm run frontend:dev

启动成功后，前端会显示类似内容：

VITE ready
Local: http://localhost:3000/

前端访问地址：
http://localhost:3000/

四、后台管理登录

打开：
http://localhost:8000/hadmin/login.html

管理员账号和密码读取 backend\.env 中的配置：

ADMIN_BOOTSTRAP_USERNAME=admin
ADMIN_BOOTSTRAP_PASSWORD=你的密码

注意：如果 backend\.env 中同一个变量出现多次，通常最后一次生效。登录失败时，优先检查文件最后面的 ADMIN_BOOTSTRAP_PASSWORD。

五、正常关闭项目

分别回到前端和后端的 PowerShell 窗口，按：

Ctrl + C

如果提示是否终止批处理操作，输入：

Y

六、常见问题

1. 后端提示 MySQL 连接失败
   先确认小皮 MySQL 已启动，再检查 backend\.env 中的 MYSQL_USER、MYSQL_PASSWORD、MYSQL_DATABASE 是否正确。

2. 前端能打开，但接口请求失败
   确认后端 http://localhost:8000 已启动，并检查 vue-project 中接口配置是否指向 localhost:8000。

3. 端口被占用
   后端默认端口是 8000，前端默认端口是 3000。
   可以用下面命令检查端口：
   netstat -ano | findstr ":8000"
   netstat -ano | findstr ":3000"

4. 修改代码后是否需要重启
   前端 Vite 通常会自动刷新。
   后端使用 nodemon，保存 backend\app.js 后通常会自动重启。

5. 后台管理页面打不开
   后台管理页面由后端服务托管，必须先启动后端。
   正确地址是：
   http://localhost:8000/hadmin/login.html

七、推荐日常启动顺序

1. 打开小皮，启动 MySQL。
2. 启动后端：
   npm run backend:dev
3. 启动前端：
   npm run frontend:dev
4. 浏览器访问：
   http://localhost:3000/
5. 后台管理访问：
   http://localhost:8000/hadmin/login.html
