# PWA 浏览器兼容性与安装指南

## 为什么有些浏览器没有“安装”按钮？

PWA (Progressive Web App) 的安装功能依赖于浏览器的支持程度。虽然我们的代码已经配置了 PWA 标准，但不同浏览器的行为和用户界面有所不同：

### 1. Google Chrome / Edge (桌面端)
- **支持最好**：在地址栏右侧会直接显示一个“安装”图标（通常是一个带箭头的显示器图标）。
- **操作**：点击图标 -> 点击“安装”。

### 2. Safari (macOS 桌面端)
- **受限支持**：Safari 桌面版目前**不支持**直接安装 PWA 为桌面应用（不像 Chrome 那样）。
- **替代方案**：您可以将网页“添加到 Dock” (macOS Sonoma 及以上版本支持更好)。

### 3. Safari (iOS 移动端)
- **支持**：但没有弹窗提示。
- **操作**：点击底部的“分享”按钮 -> 向下滑动 -> 选择“添加到主屏幕”。

### 4. Firefox (桌面端)
- **不支持**：Firefox 桌面版已经停止了对 PWA 安装（SSB）的支持。它推荐使用书签。
- **替代方案**：只能作为普通网页访问，无法安装为独立应用。

### 5. 微信 / QQ 等内置浏览器
- **不支持**：这些 App 的内置浏览器通常不支持 PWA 安装标准。
- **操作**：请点击右上角菜单 -> 选择“在浏览器中打开”。

---

## 如何验证 PWA 是否生效？

即使没有“安装”按钮，只要 PWA 配置正确，**离线访问**和**资源缓存**功能依然在所有现代浏览器中生效。

您可以尝试：
1. 打开网站加载完成后。
2. 断开网络（关闭 WiFi）。
3. 刷新页面。
4. 如果页面依然能正常显示（而不是显示“无网络连接”），说明 PWA 核心功能已生效。
