# 道德经平台 2.0 (DaoDeJing Platform 2.0)

这是一个基于 Vue 3 + TypeScript + Tailwind CSS 构建的现代化道德经学习与交流平台。致力于通过 AI 技术赋能传统文化，提供个性化学习路径、智能解读、共创社区和丰富的资源库。

## 🚀 快速开始 (打开方式)

### 1. 环境准备
确保您的电脑已安装 [Node.js](https://nodejs.org/) (推荐 v16+ 版本)。

### 2. 安装依赖
在项目根目录 (`vue-project`) 下打开终端，运行：
```bash
npm install
```

### 3. 启动项目
依赖安装完成后，运行以下命令启动本地开发服务器：
```bash
npm run dev
```

### 4. 访问页面
启动成功后，终端会显示访问地址，通常为：
[http://localhost:3000](http://localhost:3000)
在浏览器中打开该地址即可访问平台。

## 📖 操作流程指南

### 1. 🏠 首页 (Home)
- **概览**：展示平台核心价值、每日一句、功能入口。
- **导航**：通过顶部导航栏快速跳转至各功能模块。
- **登录/注册**：点击右上角“登录/注册”按钮，使用模拟账号登录（无需真实注册，点击登录即可进入已登录状态）。

### 2. 🤖 AI 智能解读 (AI Interpretation)
*路径：导航栏 -> AI解读*
- **提问**：
  - 在底部输入框输入关于《道德经》的问题。
  - 或点击界面上的“推荐问题”快速提问。
- **章节选择**：点击左上角章节选择器，指定特定章节进行解读。
- **历史记录**：
  - 点击左侧侧边栏图标（移动端）或直接在左侧查看（桌面端）。
  - 支持**新建对话**、**切换历史会话**、**删除会话**。
- **Markdown支持**：AI 回复支持 Markdown 格式，代码块和列表显示清晰。

### 3. 🛣️ 学习路径 (Learning Path)
*路径：导航栏 -> 学习路径*
- **选择路径**：根据自身水平选择 **初学者**、**进阶者** 或 **研究者** 路径。
- **学习课程**：
  - 点击“开始学习”或“继续学习”进入当前课程。
  - 查看课程进度环和统计数据。
- **课程测验**：
  - 完成课程学习后，点击“继续学习”触发测验。
  - 回答正确后自动解锁下一课，并更新进度。
  - 已完成的课程可以点击“复习”重新答题。
- **成就系统**：随着课程完成数量增加，自动解锁“初学者”、“思考者”等成就徽章。

### 4. 👥 共创社区 (Community)
*路径：导航栏 -> 共创社区*
- **浏览帖子**：
  - 通过顶部标签切换：最新发布、热门讨论、关注的人、我的收藏、待解答。
  - 使用搜索栏按标题、内容或标签查找帖子。
- **发布内容**：
  - 在顶部发帖框输入标题、内容和标签。
  - 点击 **发布帖子** 直接发布。
  - 点击 **保存草稿** 将内容存入草稿箱。
- **草稿箱管理**：点击“草稿箱”按钮查看、加载或删除未发布的草稿。
- **互动**：支持对帖子进行 **点赞**、**收藏**、**关注作者** 以及 **分享**（模拟）。

### 5. 📚 资源库 (Resource Library)
*路径：导航栏 -> 资源库*
- **查找资源**：
  - 使用顶部大搜索框或侧边栏搜索框进行全文检索。
- **高级筛选**：
  - 在侧边栏按 **资源类型** (PDF/视频/音频等)、**时代**、**评论者** 进行组合筛选。
- **排序与查看**：
  - 支持按 **最新发布**、**下载量**、**评分** 排序。
  - 点击资源卡片查看详细信息（目前为展示）。
- **分页**：底部页码条支持翻页浏览。

## 🛠️ 技术栈
- **框架**: Vue 3 (Composition API)
- **语言**: TypeScript
- **构建工具**: Vite
- **样式**: Tailwind CSS
- **路由**: Vue Router
- **状态管理**: Pinia (部分使用 Reactivity API)
- **图标**: Font Awesome

## 📁 项目结构
```
src/
├── components/   # 公共组件 (Auth, Chat, Layout, Learning...)
├── views/        # 页面视图 (Home, AI, Community, Learning, Resources...)
├── router/       # 路由配置
├── stores/       # 状态管理
├── assets/       # 静态资源
└── types/        # TypeScript 类型定义
```
