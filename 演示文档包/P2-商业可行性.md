## 任务 1 · 价值仪表盘 / 修行账单

### 评委痛点
"用户花 ¥19/月,他真能感知到值吗?后台数据(ai_usage_logs)写满了,但**用户自己看不见**。"

### 解决方案

登录用户 Home 顶部新增「我的修行账单」卡片,**4 块数据**让付费 ROI 一目了然:

| 卡片 | 数据 | 心理锚点 |
|------|------|---------|
| 本月 AI 对话 | calls 次数 + 累计 | 用得越多越值 |
| 本月 Tokens | 总 tokens / 1万 / 1k | 实际算力消耗可视化 |
| **节省咨询费** | **¥{calls × 30}** | **核心 ROI:对标心理咨询 ¥300/小时 × 6 分钟/次 ≈ ¥30/次** |
| 道学进度 | learned/23 章 + 环形进度 | 学习留存 + 复购理由 |

### 技术实现

```
GET /api/user/value-report (authMiddleware)
   └─ 3 路并行 SQL 聚合(MySQL DATE_FORMAT(NOW(),'%Y-%m-01'))
       ├─ 当月:COUNT(*) calls, SUM(total_tokens), SUM(estimated_cost)
       ├─ 累计:同上去掉日期条件
       └─ 学习进度:COUNT(DISTINCT chapter_id) FROM learning_progress WHERE completed=1
   └─ formatValueReport(rows, progress)
       · savedAdvisory = calls × ¥30(rate 系数可注入,后续 A/B)
       · progress.percent = Math.min(100, Math.round(learned/23 × 100))
   └─ 前端 5 分钟 localStorage 缓存(valueReportService.ts)
```

### 关键文件

- `backend/lib/value-report-utils.js` — 纯函数 (4 个 export)
- `backend/tests/value-report-utils.test.js` — 10 单测(constants / saved advisory 4 例 / progress 4 例 / format 2 例)
- `backend/app.js` — `GET /api/user/value-report` 路由(authMiddleware,3 并行 SQL)
- `vue-project/src/services/valueReportService.ts` — 类型化 service + 5 分钟缓存
- `vue-project/src/components/profile/ValueReportCard.vue` — 4 卡 + SVG 圆环
- `vue-project/src/views/HomeView.vue` — 集成位置:每日道签 ↓ ValueReportCard ↓ 常用入口

### 演示步骤(30 秒)

1. 登录任意用户 → 首页头部出现「我的修行账单」
2. 当月 AI 对话:**N 次** · Tokens:**X.Xk** · **节省 ¥{N×30}** · 道学进度:**M%**
3. 圆环进度从 0 → 实际百分比丝滑动画
4. 点"刷新"按钮 → 强制 force=true 重查,数字立即更新

## 任务 2 · B 端 / 团队订阅入口

### 评委痛点
"3 档 C 端订阅最高 ¥49/月,**怎么扩到企业市场?** 销售漏斗在哪?"

### 解决方案

`PricingModal.vue` 从 3 档 C 端扩到 **5 档 2 行布局**:

```
┌─────────────────────────────────────────────┐
│              个人方案(C 端)                  │
├─────────────┬─────────────┬─────────────────┤
│  布衣 ¥0    │ 居士 ¥19 ★  │   宗师 ¥49      │
└─────────────┴─────────────┴─────────────────┘

┌─────────────────────────────────────────────┐
│           企业 / 团队方案(B 端)              │
├─────────────────────┬───────────────────────┤
│  团队 ¥299/10席 NEW │  企业 联系洽谈 定制    │
└─────────────────────┴───────────────────────┘
```

- **Team** ¥299/月 · 10 席位 — 中小企业,Pro 全部权益 + 团队仪表盘 + 优先支持。点击 → 沿用 mock upgrade,后端 tier='team' 落库
- **Enterprise** 联系洽谈 — 无限席位 + 私有部署 + 定制 Persona + Open API。点击 → 打开 `EnterpriseContactModal` 表单 → `POST /api/leads` → 写 leads 表 → 24 小时内联系您 toast

### 技术实现

```
backend/lib/leads-utils.js  纯校验(易单测)
   ├─ EMAIL_REGEX / PHONE_REGEX / TEAM_SIZE_OPTIONS / INTENT_OPTIONS
   ├─ NOTE_MAX_LENGTH=1000 / NAME_MAX_LENGTH=100
   ├─ validateLeadInput(input) → { valid, errors }
   └─ normalizeLeadInput(input) → trim / 小写 email / 默认 intent='enterprise'

backend/app.js
   ├─ CREATE TABLE leads (idempotent IF NOT EXISTS)
   ├─ POST /api/leads — 匿名可填,validate + normalize + INSERT,返回 {success, id}
   ├─ GET  /api/admin/leads — adminAuthMiddleware,分页 + status 过滤
   └─ PATCH /api/admin/leads/:id — 改 status(new/contacted/qualified/closed)
   └─ validTiers 白名单加 'team'

vue-project/src/stores/auth.ts
   ├─ SubscriptionTier 加 'team'
   └─ checkLimit limits: { free:5, pro:50, team:500, master:9999 }
```

### 关键文件

- `backend/lib/leads-utils.js` — 7 export(EMAIL_REGEX / PHONE_REGEX / TEAM_SIZE_OPTIONS / INTENT_OPTIONS / NOTE_MAX_LENGTH / validateLeadInput / normalizeLeadInput)
- `backend/tests/leads-utils.test.js` — 10 单测(7 校验路径 + 安全降级 + normalize + 常量)
- `backend/app.js` — CREATE TABLE leads + 3 路由 + 'team' tier
- `vue-project/src/components/auth/PricingModal.vue` — 5 档 2 行布局,max-w-5xl → 6xl
- `vue-project/src/components/auth/EnterpriseContactModal.vue` — 表单 + POST /api/leads + 成功状态
- `vue-project/src/stores/auth.ts` — 'team' 类型 + 500 次/日额度

- **subscription_tier='team' 兼容** — 已 grep 全代码库未发现 ['free','pro','master'] 硬编码白名单(除了已扩展的 validTiers / checkLimit)
