# 演示检查清单

> 录制视频前，请逐项检查确保一切准备就绪。

---

## ✅ 一、环境准备

### 1.1 项目启动
- [ ] 运行 `npm run dev` 启动开发服务器
- [ ] 确认项目运行在 http://localhost:3000/ 或 http://localhost:3001/
- [ ] 测试首页能正常访问
- [ ] 测试各功能页面能正常打开

### 1.2 音频测试
- [ ] 确认 `video.mp3` 文件存在于 `public/` 目录
- [ ] 测试音频文件能正常播放
- [ ] 调整音量到合适水平
- [ ] 准备耳机或确认扬声器正常工作

### 1.3 网络检查
- [ ] 确认网络连接稳定
- [ ] 测试 DeepSeek API 能正常响应（可选）
- [ ] 关闭可能占用带宽的应用

---

## ✅ 二、浏览器准备

### 2.1 浏览器设置
- [ ] 使用 Chrome 或 Edge 浏览器
- [ ] 清理浏览器缓存（Ctrl+Shift+Delete）
- [ ] 关闭无关的标签页
- [ ] 关闭浏览器插件（广告拦截等）
- [ ] 设置浏览器缩放为 100%

### 2.2 开发者工具
- [ ] 关闭开发者工具（F12）
- [ ] 确保控制台无报错信息

---

## ✅ 三、录制准备

### 3.1 录制软件
- [ ] 打开屏幕录制软件
- [ ] 设置录制区域为 1920x1080
- [ ] 设置帧率为 30fps
- [ ] 测试录制功能正常
- [ ] 确认保存路径和格式

### 3.2 音频设置
- [ ] 测试麦克风正常
- [ ] 调整麦克风音量
- [ ] 关闭环境噪音
- [ ] 准备讲解稿（纸质或电子版）

### 3.3 鼠标设置
- [ ] 开启鼠标高亮（如使用）
- [ ] 调整鼠标速度适中
- [ ] 确保鼠标指针清晰可见

---

## ✅ 六、应急准备

### 6.1 备用方案
- [ ] 如果 AI 回复慢，准备截图备用
- [ ] 如果音频播放失败，准备录屏片段备用
- [ ] 如果页面加载慢，准备提前打开页面

### 6.2 问题处理
- [ ] 页面卡顿：刷新页面
- [ ] 音频不播放：检查文件路径
- [ ] AI 无响应：跳过该部分

## ✅ 七、录制流程

### 7.1 录制前（5分钟）
1. [ ] 深呼吸，放松心态
2. [ ] 测试麦克风和录制软件
3. [ ] 打开项目首页
4. [ ] 准备好讲解稿
5. [ ] 关闭手机通知

### 7.2 录制中
1. [ ] 按照脚本顺序演示
2. [ ] 保持语速适中
3. [ ] 鼠标移动要明显
4. [ ] 关键操作适当停顿
5. [ ] 出错不要慌张，可以重录

### 7.3 录制后
1. [ ] 检查视频是否完整
2. [ ] 检查音频是否清晰
3. [ ] 检查画面是否流畅
4. [ ] 备份原始录制文件
